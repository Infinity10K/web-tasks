# Домашнее задание 1

## 1. Верстка статического сайта

Цель домашнего задания — подготовить статическую верстку сайта (так называемую «рыбу») и оформить результат как полноценный учебный проект в Git. Верстку можно размещать в обычных HTML-файлах в директории public. Также можно сразу размещать верстку в шаблонах Django. Во втором случае понадобится самостоятельно создать примитивные view и настроить роутинг в urls.py. Подробнее работа с роутингом и шаблонизацией обсуждается в ДЗ2.

## 2. Срок сдачи
ДЗ1 и ДЗ2 нужно сдать к семинару, на котором выдается ДЗ3. Работы, отправленные на проверку позже, оцениваются в половину максимальных баллов.

До начала занятия, на котором выдается ДЗ3, необходимо:
- пройти ревью у ментора (в формате PR в репозитории) и внести правки;
- передать работу на финальную проверку преподавателям.

## 3. Критерии проверки

### 3.0. Репозиторий и процесс сдачи

Репозиторий должен быть создан на GitHub и назван согласно формату <год>-VK-EDU-Web-<семестр><номер учебной группы>-<фамилия>-<первая буква имени> (например, 2026-VK-EDU-Web-11-Morev-M).

Менторы и преподаватели должны быть приглашены в репозиторий как контрибьюторы с правами на просмотр и комментирование (распределение студентов по менторам и преподавателям будет зафиксировано в топике на портале).

В репозитории нужно настроить protected main branch:
- прямые push в master/main запрещены;
- merge — только через Pull Request;
- (желательно) включено требование хотя бы одного review перед merge.

Код в main должен вливаться через PR:
- для каждого ДЗ — отдельный PR с названием hw-<номер ДЗ> (например, hw-1);
- PR содержит осмысленное описание (что сделано).

Репозиторий должен содержать файл README:
- как запустить/открыть верстку (если это чистый HTML — как открыть index.html; если Django — как запустить сервер);
- перечислены страницы/роуты (index, question, ask, login, signup).

Репозиторий должен содержать файл .gitignore:
- не коммитятся мусорные файлы ОС/IDE (.idea/, .vscode/, __pycache__/, .DS_Store, *.log, *.sqlite3 и т. п.);
- не коммитятся зависимости и артефакты сборки;
- в будущем файл будет расширяться.

### 3.1. Структура репозитория

Все сверстанные моки (base.html, index.html, question.html, ask.html, login.html, signup.html и т. д.) должны лежать в отдельной папке public/.

Вся статика должна лежать локально в проекте, в public/static/, и подключаться не через CDN:
- Bootstrap CSS и собственные CSS-файлы — в public/static/css/;
- Bootstrap JS и собственные JS-файлы — в public/static/js/;
- изображения, аватарки и иконки — в public/static/img/.

### 3.2. Верстка общего вида (layout) страницы

В файле **base.html** нужно создать основную верстку (любой) страницы. Для упрощения задачи нужно скачать и использовать CSS библиотеку 
[Bootstrap](https://getbootstrap.com/docs/5.3/getting-started/introduction/).

При верстке страниц важно обратить внимание на:
- Структуру страницы: header + основной контент (центральная колонка main + правая колонка sidebar) + footer.
- Отступы и выравнивание: нет «слипшихся» блоков, адекватные margin/padding.
- Header: поисковая строка, логотип (логотип должен вести на главную страницу), юзерблок в шапке (для авторизованного на странице профиля и неавторизованного на остальных страницах), кнопка создания вопроса (переход на страницу создания вопроса).
- Sidebar: блоки Popular tags и Best members.
- Footer: прибит к низу viewport, если размер контента меньше высоты viewport.
- Title и favicon: осмысленный title страниц и проработанный favicon сайта.

Нужно проработать минимальный адаптив для ширины примерно ≥1200px, 768px, 375px:
- Header не ломается: логотип/поиск/юзерблок не наезжают друг на друга.
- Колонки: на узких экранах правая колонка уходит вверх или вниз.
- Основной контент читается без горизонтального скролла.
- Карточки вопросов/ответов: аватар/кнопки/текст не «разъезжаются», кнопки не выходят за границы.
- Формы: в мобильной версии поля растягиваются по ширине экрана, кнопки не обрезаются.

### 3.3. Верстка списка вопросов на главной странице
В файле **index.html** нужно создать верстку для списка вопросов. Основную верстку (layout) можно просто скопировать из **base.html**. При использовании шаблонизатора - вам необходимо унаследовать ([extends](https://docs.djangoproject.com/en/5.0/ref/templates/builtins/#std:templatetag-extends)) шаблон от базового. Страницы **hot.html** и **tag.html** можно проработать в ДЗ2.

Обратить внимание на:
- Терпимые отступы (padding/margin) между блоками.
- Карточку вопроса: аватарка автора, заголовок вопроса (переход на страницу вопроса), краткий текст/описание, количество ответов, теги (переход на страницу списка вопросов по тегу), время создания вопроса.
- Кнопки лайков: счётчик и три состояния виджета (нажат лайк, нажат дизлайк, ничего не нажато).
- Пагинатор вопросов: список номеров страниц и состояние текущей активной страницы.

### 3.4. Верстка страницы одного вопроса
HTML разместить в файле **question.html**.

Обратить внимание на:
- Терпимые отступы (padding/margin) между блоками.
- Карточку вопроса: аватарка автора, заголовок вопроса, краткий текст/описание, количество ответов, теги (переход на страницу списка вопросов по тегу), время создания вопроса, виджет лайков.
- Карточку ответа: аватарка автора, текст ответа, время создания ответа, виджет лайков, виджет «правильного ответа» и два состояния виджета (ответ правильный, ничего не нажато).
- Пагинатор ответов: список номеров страниц и состояние текущей активной страницы.

### 3.5. Верстка форм: добавления вопроса, форм логина и регистрации, формы профиля
HTML разместить в файлах **ask.html**, **login.html**, **signup.html**, **profile.html** соответственно.

Обратить внимание на:
- Формы должны быть сверстаны аккуратно и продуманно (например, на странице логина с переходом к регистрации по ссылке).
- Поля должны иметь адекватную ширину (и максимальную длину), выравнивание и отступы.
- В формах должны быть проработаны сообщения об ошибках формы (например, на странице логина "Вы ввели неправильный email или пароль" над формой), так и об ошибках полей (например, на странице регистрации "Вы ввели невалидный email" под полем с email).
- В формах должны быть проработаны подсказки к полям (например, на странице добавления вопроса про максимальное количество символов).

## 5. Баллы

### Максимальные баллы за ДЗ - 16 баллов

Репозиторий, процесс сдачи — 4 балла
- Правильное название + приглашены менторы/преподаватели — 1
- Protected main branch + merge только через PR — 1
- Корректный .gitignore + нет лишних файлов в репозитории — 1
- Структура проекта: отдельная папка для HTML + отдельные папки для CSS/JS/img + статика хранится локально — 1

Общий layout (base.html) — 3 балла
- Общая структура: header / main / footer / правая колонка (sidebar) с блоками в main — 1
- Header: логотип + поиск + юзерблок (состояния auth/unauth) + кнопка создания вопроса — 1
- Favicon + осмысленный title — 1

Листинг вопросов (index.html) — 2 балла
- Карточка вопроса: отступы, содержание, базовая композиция, виджет лайков — 1
- Блок с пагинацией и проработанные состояния — 1

Страница вопроса (question.html) — 3 балла
- Блок вопроса: отступы, содержание, базовая композиция, виджет лайков — 1
- Список ответов: отступы, содержание, базовая композиция, виджет «правильного ответа» + пагинация ответов — 1
- Форма ответа — 1

Верстка форм (ask.html / login.html / signup.html / profile.html) — 3 балла
- ask, login, signup: аккуратная и продуманная верстка полей, проработаны ошибки полей и формы, подсказки — 2
- profile: аккуратная и продуманная верстка полей, блок с аватаркой пользователя, проработаны ошибки полей и формы, подсказки — 1

Минимальный адаптив — 1 балл
- Корректная перестройка сетки и читаемость на мобильной ширине (без горизонтального скролла; header/формы/карточки не ломаются) — 1
